Conceptos Fundamentales de Organización de Memoria

1. Documentación Científica

Definición: Una estructura de datos es una forma sistemática y especializada de organizar, almacenar y manipular datos en la memoria de una computadora, de tal manera que puedan ser utilizados de forma eficiente según las necesidades del algoritmo.


Utilidad: El uso de una estructura adecuada optimiza los recursos de hardware, reduciendo drásticamente el consumo de memoria RAM y minimizando el tiempo de ejecución (procesamiento) de cualquier software.


Características clave:

  • Abstracción: Enfocadas en el "qué hace" la estructura antes del "cómo lo hace" internamente.
  • Lineales vs No Lineales: Los datos pueden organizarse de forma secuencial consecutiva (como arreglos y listas) o mediante relaciones jerárquicas complejas (como árboles y grafos).
  • Dinámicas vs Estáticas: Capacidad de alterar su tamaño en memoria en tiempo de ejecución.

Casos de Uso reales: El sistema de archivos de un sistema operativo (organizado en árboles), el historial de navegación de un navegador web o la cola de impresión de una oficina.

2. Ejemplos Conceptuales Solucionados

Ejemplo 1: Estructura de un Tipo de Dato Abstracto (TDA)

Modelado básico en Java de una entidad u objeto contenedor que sirve como base conceptual para estructurar información compleja:

public class Estudiante {
    String nombre;
    int codigo;
    double notaFinal;

    // Constructor que define la estructura del dato en memoria
    public Estudiante(String nombre, int codigo, double notaFinal) {
        this.nombre = nombre;
        this.codigo = codigo;
        this.notaFinal = notaFinal;
    }
}

Ejemplo 2: Intercambio de variables temporales

Lógica elemental algorítmica para reorganizar datos básicos usando una variable auxiliar en memoria:

int variableA = 50;
int variableB = 100;

// Proceso de intercambio (Swap)
int auxiliar = variableA;
variableA = variableB;
variableB = auxiliar;

System.out.println("Variable A final: " + variableA); // Imprime 100
